Agentes
O que é um Agente?
Um agente é um sistema que usa um LLM como "cérebro" para perceber o ambiente, tomar decisões e executar ações de forma autônoma para atingir um objetivo.
  • Percebe o ambiente — lê inputs, resultados de tools, contexto
  • Raciocina e planeja — o LLM decide o que fazer a seguir
  • Age no mundo — executa ferramentas, escreve, busca, calcula

Tools
O que são Tools?
Tools são funções externas que o LLM pode invocar para interagir com o mundo real — ir além do que está no seu treinamento.
🔍 Busca na web
🧮 Calculadora
🗄️ Banco de dados
🌤 API de clima
📧 Enviar e-mail
💻 Executar código
📁 Sistema de arquivos
📅 Calendário
💳 Pagamentos
Sem tools, o LLM apenas fala sobre o mundo.
Com tools, o LLM age no mundo.
Tools
Sem Tools vs Com Tools
❌ LLM sem tools
"Qual é a temperatura em Recife agora?"
🧠 LLM responde com dados do treinamento
↓
"Em Recife, as temperaturas típicas em abril ficam em torno de 28°C..."
⚠️ Dado possivelmente desatualizado
✅ LLM com tools
"Qual é a temperatura em Recife agora?"
🧠 LLM decide usar a tool get_weather
↓
get_weather(city="Recife")
→ { "temp": 31, "cond": "Sol" }
↓
"Agora em Recife está 31°C com sol!"
✅ Dado real e atualizado
Tools
Como funciona? Function Calling
1. Usuário envia pergunta ao agente
↓
2. LLM decide qual tool usar e gera parâmetros
↓
3. Sistema executa a função localmente
↓
4. Resultado retorna ao LLM como contexto
↓
5. LLM formula a resposta final
Exemplo de tool call gerado pelo LLM:
{
  "tool": "get_weather",
  "params": {
    "city": "Recife",
    "unit": "celsius"
  }
}
Resultado retornado:
{
  "temp": 31,
  "condition": "Ensolarado",
  "humidity": 72
}

MCP
Model Context Protocol
Criado pela Anthropic — 2024
Protocolo aberto e padronizado que define como LLMs se conectam a ferramentas e fontes de dados externas.
💡 O MCP é para LLMs o que o USB é para dispositivos:
um padrão universal de conexão.
📐
Padronização
Um protocolo, qualquer LLM e qualquer tool
🔒
Segurança
Controle explícito sobre o que o LLM pode acessar
♻️
Reusabilidade
Escreva um MCP Server, use em qualquer host

MCP
MCP Client vs MCP Server
🔌 MCP Client
  • Roda dentro do host / aplicação
  • Faz requisições aos servidores
  • Mantém uma conexão por servidor
  • Exemplos: Claude Desktop, Cursor
⚙️ MCP Server
  • Expõe capacidades ao LLM
  • Responde requisições do client
  • Executa operações reais
  • Exemplos: server de filesystem, GitHub
Tipos de recursos que um MCP Server pode expor:
🔧 Tools (funções executáveis)
📄 Resources (dados/arquivos)
📝 Prompts (templates)

{

👉 Sem escrever nenhum código de integração — só configurar o servidor.
A2A
Agent-to-Agent Protocol
Criado pelo Google — 2025
Protocolo que permite que agentes de IA se comuniquem e colaborem entre si, independente de qual empresa ou framework os criou.
🔑 Diferença chave:

MCP conecta   LLM ↔ Ferramentas
A2A conecta   Agente ↔ Agente
  • Agentes anunciam suas capacidades via Agent Card
  • Comunicação via HTTP + JSON — interoperável por design
  • Suporte a tarefas longas e respostas em streaming

MCP vs A2A: quando usar?
Aspecto MCP A2A
Criado por Anthropic (2024) Google (2025)
Propósito LLM acessar ferramentas externas Agentes colaborarem entre si
Quem se conecta LLM ↔ Tool / Dado Agente ↔ Agente
Quando usar Preciso que o LLM acesse filesystem, banco de dados, APIs Preciso de múltiplos agentes especializados colaborando
São exclusivos? Não — podem (e devem) ser usados juntos!
Extras
Tipos de Memória em Agentes
📝
In-Context (curto prazo)
Informação no prompt atual. Rápida, mas temporária — desaparece ao fim da sessão.
💾
Externa (longo prazo)
Banco de dados vetorial ou SQL. Persistente entre sessões. Recuperada via busca semântica.
📖
Episódica
Histórico de conversas e interações anteriores. Permite ao agente lembrar de usuários e preferências.
📚
Semântica
Conhecimento de domínio: fatos, regras, documentos. Base de conhecimento consultada via RAG.
Extras
Sistemas Multi-Agente
Por que usar múltiplos agentes?
  • Especialização: cada agente focado em uma tarefa específica
  • Paralelismo: agentes trabalhando simultaneamente, mais rápido
  • Confiabilidade: agentes verificam o trabalho uns dos outros
Exemplo: "Gere um relatório de mercado sobre IA no Brasil"

🔍 Agente Pesquisador  →  busca e coleta dados
📊 Agente Analista  →  processa e extrai insights
✍️ Agente Redator  →  escreve o relatório final
🔎 Agente Revisor  →  verifica fatos e qualidade
